# MMT-DPI — externally-updatable port-hint attribution (M9, issues #26 & #74)
#
# Point the MMT_DPI_PORT_MAP_FILE environment variable at a copy of this file to
# add L4-port -> protocol HINTS without recompiling. By default a rule is
# consulted ONLY AFTER the bundled, compiled-in port switch returns no match, so
# it never overrides a built-in mapping; when the variable is unset, nothing
# extra is loaded.
#
# Port-based attribution is the WEAKEST signal in the pipeline: it is a hint of
# last resort, used only after payload-signature and IP-range classification
# have already failed, and only when the handler opts in via
# enable_port_classify() (which is OFF by default). Treat these as guesses to be
# confirmed by a payload signature, not as authoritative classifications.
#
# Format (one rule per line):
#     <tcp|udp>  <port>  <PROTOCOL>  [override]
#   - text after '#' is a comment; blank lines are ignored
#   - <port> is in [1, 65535]
#   - <PROTOCOL> is a registered protocol name or a bare numeric protocol id
#   - the optional trailing 'override' keyword (issue #74) makes the hint be
#     consulted BEFORE the built-in switch, so it can replace a built-in mapping

# tcp 8443 SSL
# udp 5353 DNS
# tcp 80   HTTP_PROXY  override   # override the built-in port-80 -> HTTP mapping
